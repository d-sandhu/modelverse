# Portfolio case-study outline

- Problem: independent browser worlds without page reloads or renderer multiplication.
- Constraints: mobile GPU budgets, AI-authored package isolation, local-first reliability.
- Decision: shell-owned infrastructure with a deliberately narrow World SDK.
- Current proof: a world-independent renderer, registry, lifecycle contracts, local persistence, API, mobile primitives, and automated smoke harness.
- Future proof: independently authored worlds with contrasting controls, cameras, mechanics, persistence, and cleanup.
- Full stack: dynamic browser chunks, Express validation, PostgreSQL modeling, tests, and Caddy deployment.
- Evidence still required: measured performance, final screenshots, accessibility field testing, and production telemetry.
