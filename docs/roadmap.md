# Roadmap

1. Keep the platform-only smoke harness and package boundaries green.
2. The project owner defines proof-world briefs, then assigns each implementation to a separate model.
3. Use those worlds to stabilize lifecycle, local saves, failure recovery, mobile controls, and disposal.
4. Measure desktop and physical iOS/Android performance; tune adaptive thresholds.
5. Complete static preview assets and reduced-rate live render targets.
6. Add world-owned save migration callbacks and explicit audio activation.
7. Freeze World SDK 1.0 only after independent worlds pass the full journey and failure matrix.
